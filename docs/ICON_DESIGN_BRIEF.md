# 🎨 بريف تصميم أيقونات — منصة «تسعيرك»

> أرسل هذا الملف لمصمّم الأيقونات (Claude Design). فيه **كل التفاصيل والمواصفات** ليُسلّم أيقونات جاهزة نركّبها مباشرة.

---

## 1) السياق
- **تسعيرك**: منصّة B2B لتوريد مواد البناء في السعودية، الواجهة **عربية RTL**.
- **وين تظهر الأيقونات؟**
  1. شاشة اختيار المواد عند المقاول (قائمة **أكورديون**: كل مجموعة لها أيقونة على جنب الاسم).
  2. بطاقات **القطاعات** الستة في الأعلى.
  3. صفحة تخصّصات المورد (التصنيفات الفرعية).
- **الهوية البصرية:** كحلي `#1B2D5B` · برتقالي `#F5831F` · أبيض · خط Cairo.

---

## 2) المواصفات التقنية ⚙️ (إلزامية — حتى نركّبها بسهولة)
| العنصر | المطلوب |
|--------|---------|
| الصيغة | **SVG** |
| المقاس / الشبكة | `viewBox="0 0 24 24"` (الرسم داخل 20×20 مع هامش ~2px) |
| النمط | **خطّي (line / outline)** — أنيق ومتّسق |
| سُمك الخط | `stroke-width` بين **1.8 و 2** (موحّد لكل الأيقونات) |
| التعبئة | `fill="none"` |
| **اللون** | **`stroke="currentColor"` فقط — لون واحد، بدون أي ألوان ثابتة** (عشان نلوّنها بالكود: كحلي عادةً، برتقالي عند فتح المجموعة) |
| الزوايا | `stroke-linecap="round"` و `stroke-linejoin="round"` |
| الوزن البصري | متّسق بين كل الأيقونات (نفس الإحساس والتفاصيل) |
| **اسم الملف** | **مفتاح المجموعة بالضبط بالإنجليزي** (مثلاً `concrete.svg`، `masonry.svg`) — مهم جداً للتركيب الآلي |
| التسليم | ملفات SVG منفصلة، **أو** نسخ كود `<path …/>` لكل أيقونة في جدول مقابل المفتاح |

**مثال للشكل المطلوب (نفس روح هذا الكود):**
```svg
<svg viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.8"
     stroke-linecap="round" stroke-linejoin="round">
  <!-- مسارات الأيقونة هنا -->
</svg>
```

## 3) القطاعات الستة (أيقونات رئيسية)
| المفتاح (اسم الملف) | الاسم | فكرة الأيقونة |
|---|---|---|
| `civil` | مدني | مبنى/أساسات قيد الإنشاء |
| `architectural` | معماري | مخطّط أرضية أو عناصر تشطيب داخلي |
| `electrical` | كهرباء | برق/قابس كهرباء |
| `mechanical` | ميكانيك | ماسورة + تكييف (أو ترس وأنابيب) |
| `equipment` | آليات ومعدات | حفّارة أو شاحنة |
| `supply_store` | محل توريد | واجهة محل/متجر |

## 6) كيف نركّبها بعد التسليم
1. ضع ملفات الـSVG (أو أكواد الـpath) **مسمّاة بالمفتاح**.
2. نحدّثها داخل مكوّن واحد عندنا (`CatIcon`) — كل أيقونة `<path>` مقابل مفتاحها.
3. الألوان تلقائية من الكود (`currentColor`)، فالمصمّم **ما يحتاج يضيف ألوان**.

> الأولوية: **القطاعات الستة + مجموعات المواد (القسم 3 و 4)** أولاً، والتصنيفات الفرعية لاحقاً.
